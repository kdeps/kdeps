The Solution

AI agents in YAML.
Local-first. Deploy anywhere.

kdeps is an open-source AI agent orchestration framework. Describe your pipeline in YAML. kdeps handles execution, routing, packaging, and deployment -- from laptop to cloud to edge.

YAML-first

No Python. No boilerplate. Define agents the way you define infrastructure -- readable by any engineer on the team.

Local-first

Full GPU-accelerated inference on your hardware. Zero cloud dependency. Data never leaves your infrastructure unless you choose.

Portable

Compile agents to single .kdeps package files. Run them anywhere -- laptop, server, Docker, cloud, edge -- no environment setup.

03 / 15

The Product

Two modes. One framework.

# workflow.yaml
resources:
  - id: fetch
    type: httpClient
    httpClient:
      url: "{{ .Input.url }}"
      method: GET

  - id: summarize
    type: chat
    requires: [fetch]
    chat:
      model: llama3.2
      prompt: |
        Summarize: {{ .fetch.response.body }}

# runs offline. GPU-accelerated.
# bundles to: summarizer-1.0.0.kdeps
Workflow Mode

Deterministic Pipelines

DAG of typed resources -- HTTP client, chat, SQL, Python, bash, web scraper. Steps run in dependency order. Predictable, testable, auditable. Ideal for production pipelines.

Agent Loop Mode

Autonomous Agents

Full LLM agent loop with tool use, conversation memory, multi-agent orchestration (agencies), and REPL. Your YAML workflows become tools the LLM decides to invoke.

04 / 15

Why kdeps

Built for how teams actually ship AI

01

YAML-first -- not Python-first

Any developer reads and writes kdeps workflows. No ML framework expertise required. Ops, DevOps, and backend engineers can own agent pipelines.

02

Local GPU inference built-in

Ollama, llama.cpp (GGUF), llamafile -- all first-class backends. Run llama3.2 on an NVIDIA GPU with a single config field. No cloud required.

03

Portable packaging -- like Docker for AI agents

Agents compile to .kdeps files. Share, install, and run them like binaries. No environment setup for recipients. Agents are versioned artifacts.

04

Vendor-agnostic by design

Switch from OpenAI to Anthropic to a local model by changing one field. The same workflow runs on any provider. No rewrite required.
